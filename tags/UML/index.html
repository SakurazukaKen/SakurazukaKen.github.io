2017-06-11
  • UML

部署图是什么

部署图是系统物理架构的模型, 显示了软件和硬件组件间的关系和过程的物理分工. 通常会在开发过程的实现阶段来编写. 表现分布式系统中不容节点的物理组织关系, 每个节点存放的工件和工件所实现的组件和其他产物. 其中node可以是计算机, 传感器, 打印机等硬件设备或者其他能够提供系统运行环境的设备(类似docker或者vm), 说完了部署图是什么, 我们来介绍下部署图的主要元素

元素

Node

node是用来表示系统中计算资源的元素, 例如个人计算机, 服务器, 传感器或者vm, docker等,

node用path链接可以表示节点间的网络结构

Execution environment

运行环境是是一种用来表示特定执行环境的node, 包括操作系统和BDMS

Artifacts

工件用来表示软件系统的物理实体, 比如可执行文件, 引用库, 软件插件, 文档或者是数据库

Deployment specifications

部署规格是实际上就是部署的可配置文件, 类如一个xml文件或者文档, 用来表示工件如何部署在节点上

Association

连接表示各组件间交互的通信路径, 通常带有通信协议

例子分析

以书中图为例, 可以看到服务器dell poweredge 3400上运行有red hat系统, 在该系统之上运行着PostgreSQL 10DBMS, 另一台服务器Dell powered 3600上运行着apache 和tomcat servlet runtime 在该runtime中运行有webstore.war这个工件, tomcat与daatabase通过sql语句交互, 而客户端有一台genericPc部署工件myrichGUIClient.ext, 与apache通过soap/http协议通讯, 另一台运行着浏览器, 与apache通过http协议通讯

注意点

部署图和组件图不同, 部署图表示的是组件和工件在部署环境中的的关系, 而组件图则表示了组件和工件在软件系统中的组成关系.
